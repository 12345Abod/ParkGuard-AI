
# نظام التحكم الذاتي بالطائرات بدون طيار ومحاكاة بيئة Gazebo

مشروع متكامل لمحاكاة وتشغيل طائرة بدون طيار (Drone) ذاتية القيادة باستخدام بيئة **Gazebo Sim** ونظام الملاحة **PX4 Autopilot**، مع واجهة برمجية للتحكم بالمهام الذاتية عبر **MAVSDK-Python**.

يحتوي المشروع على بيئة مخصصة ثلاثية الأبعاد تحاكي منطقة اختبار واقعية، بالإضافة إلى سيناريو طيران آلي لزيارة نقاط جغرافية محددة مسبقاً (Waypoints) ومراقبة حالة الطائرة.

---

## 🛠️ المكونات الرئيسية

1. بيئة المحاكاة (`external_world.sdf`) 

عالم محاكاة مخصص ومتوافق مع نظام PX4 ، مجهز بالخصائص التالية:

* 
**الإحداثيات الجغرافية (GPS Home):** مضبوطة بدقة على إحداثيات موقع الاختبار لضمان دقة مستشعرات الملاحة.


* 
**المركبات والنماذج المدرجة:** المحاكاة تحتوي على نماذج لسيارات حديثة (Hyundai Elantra N, Chevrolet Malibu) وموقع هبوط (Helipad).


* 
**إعدادات الفيزياء:** معدل تحديث 250 هرتز مع خطوة زمنية قدرها 0.004 ثانية متوافق تماماً مع مستشعرات PX4.



### 2. نص التحكم الآلي (`ugugo.py`)

برنامج مكتوب بلغة Python يعتمد على البرمجة غير المتزامنة (`asyncio`) لتنفيذ المهام التالية:

* الاتصال التلقائي بالطائرة عبر بروتوكول UDP.
* التحقق من سلامة مستشعرات الـ GPS والموقع الأساسي (Home Position) قبل الإقلاع.
* فحص حالة البطارية ديناميكياً وإعطاء تحذيرات فورية حسب النسبة المتبقية.
* الإقلاع الآلي والتوجه إلى نقاط مسار محددة بدقة على ارتفاع مخصص مع التوقف (Hover) المؤقت عند كل نقطة.
* العودة التلقائية إلى نقطة الانطلاق (RTL) والهبوط بأمان بعد اكتمال المهمة.

---

## 🚀 طريقة التشغيل والإعداد

المشروع مصمم ليعمل على نظام **Ubuntu** المجهز مسبقاً ببيئة ROS 2 / Gazebo و PX4 Autopilot.

### الخطوة 1: إعداد ملف العالم (One-time Setup)

يجب نسخ ملف العالم المخصص إلى دليل العوالم الخاص بـ PX4 لتتمكن البيئة من التعرف عليه:

```bash
cp external_world.sdf ~/PX4-Autopilot/Tools/simulation/gz/worlds/external_world.sdf

```

### الخطوة 2: تشغيل محاكي Gazebo

افتح نافذة ترميز جديدة (**Terminal 1**) وقم بتهيئة المتغيرات البيئية ثم تشغيل المحاكي:

```bash
export GZ_SIM_RESOURCE_PATH=~/PX4-Autopilot/Tools/simulation/gz/models
export GZ_SIM_SERVER_CONFIG_PATH=~/.simulation-gazebo/server.config
export GZ_SIM_SYSTEM_PLUGIN_PATH=~/PX4-Autopilot/build/px4_sitl_default/src/modules/simulation/gz_plugins
export GZ_MESH_FORCE_ASSIMP=1

gz sim -r ~/PX4-Autopilot/Tools/simulation/gz/worlds/external_world.sdf

```

### الخطوة 3: تشغيل طائرة PX4 (SITL)

بعد تشغيل واجهة Gazebo، افتح نافذة ترميز ثانية (**Terminal 2**) لتشغيل برمجية الطائرة واقترانها بالعالم المفتوح:

```bash
cd ~/PX4-Autopilot
PX4_GZ_STANDALONE=1 PX4_GZ_WORLD=external_world make px4_sitl gz_x500

```

### الخطوة 4: تشغيل كود التحكم الآلي

تأكد من تثبيت مكتبة `MAVSDK` أولاً:

```bash
pip install mavsdk

```

ثم قم بتشغيل نص الطيران الذاتي:

```bash
python3 ugugo.py

```

## 📊 مسار الطيران والمهام الذاتية

عند تشغيل الكود، ستنفذ الطائرة الخطوات الآتية بالتتابع:

1. **الاتصال وفحص الأنظمة:** الانتظار حتى اكتمال اتصال الـ UDP وتأكيد جاهزية إشارة الـ GPS.
2. **فحص البطارية:** قراءة حالة الطاقة والتأكد من ملاءمتها للطيران.
3. **الإقلاع الآلي:** الإقلاع والارتفاع حتى 10 أمتار والاستقرار في الهواء.
4. **الملاحة بين النقاط:** التوجه تلقائياً إلى النقاط الجغرافية المحددة في الكود:
* **الذروة الأولى (Waypoint 1):** `Lat: 47.398164, Lon: 8.546238`
* **الذروة الثانية (Waypoint 2):** `Lat: 47.398238, Lon: 8.546031`
* *الانتظار لمدة 5 ثوانٍ عند كل نقطة للاستقرار أو المراقبة.*


5. **العودة للهبوط:** تفعيل أمر العودة إلى نقطة الإقلاع الأصلية والتأكد من هبوط الطائرة على الأرض بسلام وإنهاء المهمة.

## 📌 ملاحظات تقنية للمطورين

* 
**إضافة مجسمات جديدة:** عند رغبتك في استيراد مجسمات أو نماذج ثلاثية أبعاد جديدة (External Meshes)، يفضل وضعها في إحداثيات تقع بين `X=10` و `X=30` ليتمكن نظام الكاميرا في الطائرة من رصدها مباشرة عند الإقلاع من نقطة الصفر.


* **إعدادات المكونات الإضافية:** لا يحتوي ملف الـ SDF على علامات `<plugin>` صريحة؛ حيث يتم تحميل ملحقات الاستشعار لـ PX4 تلقائياً عبر ملف الإعدادات الموحد `server.config`.
